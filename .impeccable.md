## Design Context

_Current working assumptions inferred from the existing product. Update this section if product direction changes._

### Users
OpenClaw Web Client is currently optimized for internal and developer-adjacent users who connect to multiple servers, inspect agent behavior, and move quickly between active conversations, history, and debugging context.

### Brand Personality
Calm, capable, lightweight. The interface should feel reliable and efficient rather than flashy, while still feeling modern and polished.

### Aesthetic Direction
Preserve the existing mobile-first chat utility language: soft green primary accents, generous rounding, clean spacing, blurred overlays, and full light/dark mode parity. Avoid neon cyberpunk, over-decorated social-chat styling, or dense enterprise-dashboard heaviness.

### Design Principles
1. Keep high-frequency actions obvious and fast.
2. Reuse established overlay and drawer patterns instead of introducing new modal styles.
3. Show connection and agent context close to content so multi-server workflows stay legible.
4. Use motion to explain panel transitions and hierarchy, not as decoration.
5. Maintain parity between light and dark themes for every new surface.
