## Design Context

### Users
Clawline is a **consumer-facing** Web chat client for OpenClaw AI agents. Primary users are general consumers who want to chat with AI assistants — not necessarily developers. The interface must be intuitive enough for non-technical users while still feeling capable for power users who manage multiple agents/servers.

**Key jobs to be done:**
- Start and manage conversations with AI agents quickly
- Switch between multiple active chats seamlessly
- Review conversation history and agent context
- Connect to different servers and manage agent relationships

### Brand Personality
**Calm · Capable · Lightweight**

The interface should feel like a trusted tool — always responsive, never overwhelming. It should inspire confidence that messages are delivered, agents are working, and nothing is lost. Think of a professional messaging app that "just works."

### Aesthetic Direction
- **Reference:** Discord — structured channel/chat layout, clean information hierarchy, comfortable for long sessions, dark mode done right, playful but not childish
- **Anti-reference:** Overly decorative social apps (WeChat Moments-style), dense enterprise dashboards, neon cyberpunk aesthetics, generic AI chat wrappers with excessive gradients
- **Visual tone:** Professional, efficient, reliable
- **Theme:** Light + Dark mode with full parity; GitHub-inspired neutral palette with `#EF5A23` orange-red as the primary accent
- **Typography:** Plus Jakarta Sans (Latin) + PingFang SC / Noto Sans SC (CJK) — clean, modern, highly legible
- **Shape language:** Large radius (24px buttons, 20px cards), iOS-style controls (toggles, sheets), glass-card overlays with backdrop blur
- **Motion:** Purposeful — panel transitions, hierarchy cues, tap feedback. Never decorative. Respect `prefers-reduced-motion`.
- **Icons:** Lucide (consistent stroke width, rounded style)

### Design Principles
1. **Instant and obvious** — High-frequency actions (send message, switch chat, search) must be reachable in ≤1 tap. No hunting.
2. **Professional confidence** — Every surface should convey reliability. Connection status, delivery ticks, loading states — users should always know what's happening.
3. **Pattern consistency** — Reuse established overlay, drawer, and sheet patterns. Don't invent new modal styles. Follow Discord's approach to progressive disclosure.
4. **Light/dark parity** — Every new component must work in both themes from day one. No afterthought dark mode.
5. **Mobile-first, desktop-ready** — Design for phone screens first (PWA), scale up gracefully. Safe-area aware, touch-target compliant (≥44px).
6. **Motion explains, not decorates** — Transitions show spatial relationships and state changes. Keep durations short (150–300ms). Skip animation entirely under `prefers-reduced-motion`.
7. **Accessible by default** — Semantic HTML, ARIA labels on interactive elements, keyboard navigable, sufficient color contrast (WCAG AA minimum).

### Component Conventions
- **Buttons:** CVA variants — default (primary fill), outline, ghost, destructive, icon. Pill-shaped (24px radius).
- **Cards:** `GlassCard` with backdrop-blur for floating surfaces (bottom nav, overlays).
- **Navigation:** Floating bottom nav bar with icon + label, safe-area offset, unread badges.
- **Overlays:** Bottom sheets and drawers (mobile pattern), not centered modals.
- **Toggles:** iOS-style custom checkbox with smooth transition.
- **Code blocks:** Catppuccin Mocha syntax highlighting with dynamic `highlight.js` language loading.
